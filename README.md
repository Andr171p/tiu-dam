# 1. Концепция и цели системы

 - Любое изображение/видео, публикуемое вовне, проходит через сервис и получает водяной знак (явный или мета-данные).
 - Система навязывает обязательное указание лицензии и источника.
 - Полный аудит: кто, когда и где использовал конкретный файл.
 - Единое API для получения «правильных» версий картинок (нужного размера, с watermark, оптимизированных).

# 2. Техническое задание

## 2.1 Пользователи и роли

| Роль | Права |
|------|-------|
| Администратор системы | Полный доступ, настройка водяных знаков, политик лицензий, моделей ML. |
| Контент-менеджер / Редактор | Загрузка, редактирование, валидация лицензий, утверждение активов. Работа в веб-редакторе. |
| Сотрудник | Загрузка в личные коллекции, запрос на публикацию. Видит только свои файлы до одобрения. |
| Дизайнер | Создание и загрузка «фирменных» материалов, настройка пресетов водяных знаков. |
| API-клиент | Системная учётная запись с read-only доступом к утверждённому контенту для динамической вставки. |

## 2.2 Функциональные требования

### 2.2.1 Модуль загрузки и индексирования

 - Загрузка через веб-интерфейс (drag-and-drop), пакетная загрузка до 50 файлов, максимальный размер: изображения до 100 МБ, видео до 4 ГБ.
 - Автоматическое извлечение метаданных: EXIF, IPTC, длительность видео, кодек, разрешение.
 - **Обязательная валидация лицензии:** при загрузке пользователь обязан указать источник (собственная съёмка, приобретённая лицензия, свободная лицензия и т.п.) и приложить подтверждение (скриншот/файл лицензии).
 - Автоматическая проверка на «чужие» водяные знаки и стоковые логотипы
 - Генерация превью: thumbnail 200x200, preview 800x600; для видео — анимированный превью (GIF/WEBP) и постер.

### 2.2.2 Водянные знаки (Watermarking)

 - **Статические шаблоны:** администратор создаёт несколько вариантов (полупрозрачный логотип ТИУ в углу, центральный паттерн, текст «© ТИУ, 2026»).
 - **Динамический водяной знак:** наложение при скачивании/отдаче через API информации о дате, пользователе, ID актива (незаметный стеганографический слой или явная подпись в углу) — для отслеживания «утечек».
 - **Пакетное наложение:** изменение водяного знака на всех ранее обработанных файлах при смене фирменного стиля.
 - **Настройка:** прозрачность, позиция, размер, шрифт (подгружается фирменный шрифт ТИУ), адаптация под альбомную/книжную ориентацию.

### 2.2.3 Изменение размера (Resize)

 - **Пресеты:** администратор задаёт именованные форматы (например, news_preview, social_instagram, print_a4).
 - **Умная обрезка (smart crop)**: на основе AI (детекция лиц/объектов) — автоматически центрирует важную часть сюжета.
 - **Вызов через API:** **GET** `/api/v1/assets/{id}/rendition?preset=news_preview&watermark=standard` возвращает готовую картинку.
 - **On-the-fly трансформация:** результат кешируется на уровне CDN.

 ### 2.2.4 Редактор изображений (Web-based)

 - Встроенный в браузер редактор
 - **Базовые операции:** кроп, поворот, зеркалирование, коррекция яркости/контрастности.
 - Наложение текста с выбором фирменных шрифтов и цветов.
 - Возможность добавлять стрелки, рамки, размытие (для замазывания лиц/чувствительных данных).
 - Сохранение как новой версии актива, без перезаписи оригинала.
 - Пункт «Проверка прав» в редакторе: кнопка быстрого запуска AI-анализа текущего изображения на наличие признаков чужих копирайтов.

### 2.2.5 Управление лицензией

 - Поле «Тип лицензии» в карточке актива: Собственная, Коммерческая (с датой окончания), Creative Commons (с указанием конкретной версии), Открытый источник (с обязательной ссылкой).
 - Автоматическое предупреждение за 30 дней до истечения коммерческой лицензии.
 - Блокировка использования актива с истекшей лицензией.
 - **Модуль аудита использования:** логирование каждого скачивания (кто и когда скачал файл).


### 2.2.6 Интеграции

 - **REST API** для Web-ресурсов: получения списка активов, поиск, скачивание нужного варианта.
 - Интеграция с ТИУ SSO для единого входа

### 2.2.7 Поиск и фильтрация

 - Полнотекстовый поиск по названию, описанию, ключевым словам
 - Фильтрация по тегам, дате, автору, и.т.д
 - Поиск по визуальному сходству (контентный поиск — «покажите все фото похожие на это»)
 - Семантический поиск (поиск по описанию медиа)

## 2.3 Нефункциональные требования

 - **Безопасность:** все изображения, отдаваемые вовне без водяного знака, запрещены на уровне API. Исходники недоступны для скачивания без специального права.
 - **Отказоустойчивость:** файлы хранятся в S3-совместимом хранилище с репликацией, метаданные — в PostgreSQL с бэкапом.
 - **Аудит действий:** каждая операция (загрузка, скачивание, редактирование) логируется с привязкой к пользователю.


# 3. Технологический стек

## 3.1 Backend

 - **FastAPI** -  в качестве основного Web фреймворка.
 - **SQLAlchemy 2.0 (async) + Alembic** — миграции, работа с PostgreSQL.
 - **PostgreSQL** — метаданные, пользователи, ACL, версии файлов.
 - **Redis** — кеширование, брокер сообщений для FastStream, Pub/Sub для событий реального времени.
 - **FastStream** — очередь задач для процессоёмких операций (генерация превью, наложение watermark, видео-транскодинг, ML-проверки).
 - **MinIO (или AWS S3)** — локальное объектное хранилище для исходных файлов и кешированных рендоров.
 - **Elasticsearch** — поисковый индекс (метаданные, визуальные эмбеддинги для поиска по сходству). (можно будет обойтись одним PostgreSQL)

## 3.2 Обработка изображений и видео

 - **Pillow** — базовая работа с изображениями, наложение текста/логотипов.
 - **Wand** (обёртка ImageMagick) — более продвинутые трансформации, работа с PDF/видео кадрами.
 - **OpenCV** — компьютерное зрение, поиск шаблонов (чужих логотипов), face detection для умной обрезки.
 - **FFmpeg** — извлечение кадров, создание превью, перекодирование видео, прожиг watermark в видео.

## 3.3 Водянные знаки

 - Кастомный пайплайн используя **Pillow/Wand**
 - **pyvips** — для больших файлов и потоковой обработки (как альтернатива)

## 3.4 Редактор

 - Toast UI Image Editor (open-source)
 - Серверная часть редатора: POST запрос принимающий применённые трансформации в формате набора команд (JSON), сервре воспроизводит операции через Pillow и сохраняет как новую версию.

## 3.5 AI/ML

### 3.5.1 Обнаружение водяных знаков

Как пример обученная модель YOLOv8 на датасете логотипов крупных фотостоков (Shutterstock, Getty Images, Adobe Stock) и других организаций. При загрузке изображения выполняется детекция; если confidence > 0.8 — актив помечается «подозрительным» и блокируется до ручной проверки.

### 3.5.2 Автоматическое определение тегов

Использование предобученной модели (ResNet, ViT) для генерации ключевых слов (интерьер, лекция, группа людей, здание), что облегчает поиск.

### 3.5.3 Визуальный поиск дубликатов

Генерация perceptual hash (pHash, dHash) для всех загружаемых изображений. Поиск похожих по расстоянию Хэмминга в специальном столбце PostgreSQL или через Elasticsearch с dense_vector. Позволяет быстро проверить, не загрузил ли кто-то то же самое изображение из интернета.

### 3.5.6 Контроль качества фото

 - Определение размытости, засветки, низкого контраста — предупреждение загружающему
 - Обнаружение лиц и объектов (OpenCV Haar cascades / YOLO) для автоматического центрирования при ресайзе.

**Примечание:** в идеале модели должны запускаться используя ONNX runtime

# 4. Основные потоки

## 4.1 Поток обработки загрузки

1. Файл через API попадает во временную папку MinIO.

2. FastAPI создаёт запись со статусом processing и отправляет задачу в очередь.

3. Worker выполняет:
 - Извлечение метаданных.
 - Анализ AI на чужие логотипы.
 - Генерацию превью и thumbnail, сохранение в MinIO.
 - Создание поискового индекса в ES.
 - Если статус одобрен (лицензия валидна, нет подозрений) – актив становится approved. Иначе — pending_review с уведомлением модераторам.
